How I think about this stuff

01

The model is rarely the bottleneck

Most production AI problems are data quality, latency, and trust issues — not model capability. I've spent more time building output validators and audit trails than tuning prompts.

02

Start with low autonomy

The prod support agent starts with workflow tools and human-in-the-loop steps. Agents that do too much too fast are hard to debug and harder to get sign-off for in regulated environments.

03

Enterprise experience matters for AI work

15 years of Java and regulated-environment deployments means I think about auditability, failure modes, and compliance from the start — not as an afterthought when something goes wrong in prod.

04

Skeptical of the hype cycle

Agents are genuinely useful. They're also fragile in ways demos don't show. I've shipped both the wins and the failure cases in production — the second teaches you more.

05

Agentic coding is a force multiplier, not a replacement

I use AI coding assistants heavily and the velocity gain is real. What makes it work is knowing what correct looks like — the right architecture, the edge cases, the things that will bite you in six months. Without that, you're just generating code faster. With it, you're compounding.
